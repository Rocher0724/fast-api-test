## 목차

### 2장. 에러는 어디에서 나는가?

에러가 발생할 수 있는 전체 그림을 먼저 이해한다.

- **1.1** 웹 서비스의 전체 구조 이해
  - 브라우저(프론트엔드) → 인터넷 → 서버(백엔드) → DB 흐름도
  - 각 구간별로 에러가 발생할 수 있다는 개념
- **1.2** 에러의 5가지 발생 지점
  - 프론트엔드 에러 (HTML/CSS/JS - 브라우저에서 발생)
  - 백엔드 에러 (FastAPI - 서버에서 발생)
  - 서버↔DB 에러 (데이터베이스 연결/쿼리 실패)
  - 프론트↔백 통신 에러 (CORS, 네트워크, API 호출)
  - 배포 환경 에러 (Render.com - 로컬에선 되는데 배포하면 안 되는 것)
- **1.3** HTTP 상태 코드 빠르게 읽기
  - 4xx: 내(프론트)가 잘못 보냄
  - 5xx: 서버(백엔드)가 잘못 처리함
  - 200: 정상 / 404: 못 찾음 / 422: 데이터 형식 오류 / 500: 서버 내부 에러

---

### 3장. 프론트엔드 에러 확인법: 브라우저 개발자 도구 

브라우저에서 에러를 확인하는 가장 기본적인 방법을 익힌다.

- **2.1** 개발자 도구(DevTools) 열기
  - F12 / Cmd+Option+I 로 여는 법
  - 꼭 알아야 할 탭 3개: Console, Network, Elements
- **2.2** Console 탭 - JS 에러 잡기
  - 빨간색 에러 메시지 읽는 법
  - `Uncaught ReferenceError`, `TypeError` 등 흔한 에러 유형
  - `console.log()`로 값 확인하는 디버깅 기법
- **2.3** Network 탭 - API 통신 에러 잡기
  - API 요청이 실제로 나갔는지 확인하기
  - 요청 URL, Method, Status Code 확인
  - Request/Response 내용 확인하기
  - 빨간색으로 표시된 실패 요청 분석

---

### 4장. 백엔드 에러 확인법: FastAPI 로그와 디버깅 

서버에서 발생하는 에러를 찾고 고치는 방법을 배운다.

- **3.1** FastAPI 서버 로그 읽기
  - 터미널에서 `uvicorn` 실행 시 나오는 로그 읽기
  - Traceback(역추적) 읽는 법 - 아래에서 위로 읽기
  - 에러 메시지에서 파일명과 줄 번호 찾기
- **3.2** FastAPI가 자주 뱉는 에러들
  - `422 Unprocessable Entity` - Request Body/Query 파라미터 불일치
  - `500 Internal Server Error` - 코드 내부 로직 에러
  - `404 Not Found` - 라우트 경로가 틀렸을 때
  - `ImportError` / `ModuleNotFoundError` - 패키지 미설치
- **3.3** /docs (Swagger UI)로 API 테스트하기
  - `http://localhost:8000/docs` 로 접속
  - API를 프론트 없이 직접 호출해서 테스트
  - 프론트 문제인지 백엔드 문제인지 분리하기
- **3.4** print() 디버깅과 에러 처리
  - 의심되는 곳에 `print()` 넣어 값 확인하기
  - `HTTPException`으로 의도적 에러 응답 만들기
  - `try-except`로 예상치 못한 에러 잡기

---

### 5장. 서버↔DB 에러: 데이터베이스 통신 문제 

서버가 데이터베이스와 통신할 때 발생하는 에러를 다룬다.

- **4.1** DB 연결 에러
  - `ConnectionRefusedError` - DB 서버가 꺼져있거나 주소가 틀림
  - 연결 문자열(DATABASE_URL) 오타 또는 누락
  - 연결 타임아웃 - DB 서버 응답이 너무 느릴 때
- **4.2** SQL / ORM 쿼리 에러
  - 테이블이 존재하지 않음 - 마이그레이션 미실행
  - 컬럼명 불일치 / 타입 불일치
  - `IntegrityError` - unique 제약조건 위반, NOT NULL 위반
- **4.3** 커넥션 풀(Connection Pool) 이해하기
  - 커넥션 풀 개념과 부족 에러
  - 올바른 연결 관리 패턴 (`Depends`)
- **4.4** 마이그레이션 관련 에러
  - 코드 변경 vs DB 반영의 차이
  - Alembic 기본 명령어
- **4.5** 배포 환경에서의 DB 에러
  - SQLite → PostgreSQL 전환 시 차이점
  - DB 무료 플랜 제한 (연결 수, 용량, 90일 만료)

---

### 6장. 프론트↔백 통신 에러: 가장 흔한 함정들 

프론트엔드와 백엔드가 통신할 때 발생하는 에러를 다룬다.

- **5.1** CORS 에러
  - "Access-Control-Allow-Origin" 에러가 뭔지
  - FastAPI에서 CORS 미들웨어 설정하는 법
  - 왜 로컬에서는 되고 배포하면 안 되는지
- **5.2** fetch() / API 호출 실수
  - URL 경로 오타 / HTTP Method 불일치
  - Request Body 형식 오류 (Content-Type 누락)
  - `async/await` 빠뜨려서 Promise 객체가 출력되는 경우
- **5.3** JSON 데이터 주고받기 실수
  - `JSON.stringify()` / `.json()` 누락
  - 서버 필드명과 프론트 필드명 불일치
- **5.4** 에러 응답 처리
  - `response.ok`로 에러 체크하는 패턴

---

### 7장. 배포 에러 확인법: Render.com 

로컬에서는 잘 되는데 Render에 배포하면 안 되는 상황을 해결한다.

- **6.1** Render 대시보드에서 로그 보기
  - Deploy 로그 vs Runtime 로그 차이
  - 빌드 실패 / 시작 실패 / 런타임 에러 구분
- **6.2** 배포 시 자주 발생하는 에러들
  - `requirements.txt` 패키지 누락, Start Command 오류
  - 환경변수 미설정, Python 버전 불일치
- **6.3** "로컬 vs 배포" 환경 차이 체크리스트
  - 대소문자, 환경변수, 패키지 버전, CORS 도메인
- **6.4** Render 무료 플랜 콜드 스타트
  - 15분 비활성 시 서버 Sleep → 첫 접속 30~60초 지연
  - 에러가 아닌 정상 동작임을 이해
- **6.5** 환경변수 관리: .env와 python-dotenv
  - 로컬(.env) vs 배포(Render Environment) 환경변수 흐름
  - python-dotenv 설정 방법
- **6.6** 알아두면 좋은 것: 실제 서버 운영
  - SSH 접속, 로그 확인, 서버 상태 체크 등 실무 맛보기

### 8장. 에러를 체계적으로 해결하는 5단계 프로세스

에러를 만났을 때 당황하지 않고 따라갈 수 있는 루틴을 만든다.

- **Step 1: 재현** - 어떤 동작에서 에러가 나는지 정확히 파악
- **Step 2: 위치 파악** - 프론트? 백? DB? 통신? 배포? (2장~6장 활용)
- **Step 3: 에러 메시지 읽기** - 에러 메시지를 끝까지 읽고 핵심 키워드 찾기
- **Step 4: 원인 탐색** - 에러 검색, AI 질문, 이분법 디버깅
- **Step 5: 수정 후 확인** - 고친 뒤 반드시 같은 동작으로 재확인
