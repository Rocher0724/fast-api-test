# 부록 C. AI에게 에러를 효과적으로 질문하는 법

---

## 왜 질문을 잘 해야 하는가?

AI에게 "에러가 나요"라고만 하면 일반적인 답변만 돌아온다. **구체적으로 질문**하면 정확한 답을 바로 받을 수 있다.

---

## 에러 질문 공식

```
[상황] + [에러 메시지] + [관련 코드] + [시도한 것]
```

이 4가지를 포함하면 AI가 바로 원인을 파악할 수 있다.

---

## 나쁜 질문 vs 좋은 질문

### 나쁜 질문

```
"에러가 나요. 어떻게 해야 하나요?"
"서버가 안 돌아가요"
"422 에러가 뭐예요?"
```

→ AI가 일반적인 설명만 할 수 있다.

### 좋은 질문

```
FastAPI 서버에서 사용자 생성 API를 호출하면 422 에러가 납니다.

[에러 메시지]
{
    "detail": [
        {"loc": ["body", "username"], "msg": "field required"},
        {"loc": ["body", "age"], "msg": "value is not a valid integer"}
    ]
}

[서버 코드] (혹은 파일명)
class UserCreate(BaseModel):
    username: str
    age: int

@app.post("/api/users")
def create_user(user: UserCreate):
    return {"message": f"{user.username} 생성 완료"}

[프론트 코드] (혹은 파일명)
fetch("/api/users", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ user_name: "홍길동", age: "스물다섯" })
});

이 에러를 어떻게 고칠 수 있나요?
```

→ AI가 "user_name을 username으로 바꾸고, age를 숫자로 보내세요"라고 바로 답한다.

## 상황별 질문 템플릿

### 템플릿 1: 에러 해결

```
[상황] FastAPI + 바닐라 HTML로 만든 웹앱에서 [어떤 기능]을 사용할 때

[에러] [에러 메시지 전체 복사 붙여넣기]

[코드]
# 관련 코드 (서버와 프론트 양쪽)

[시도한 것]
- xxx를 해봤는데 안 됐다
- yyy를 확인했는데 문제가 없었다

이 에러의 원인과 해결 방법을 알려주세요.
```

### 템플릿 2: 배포 에러

```
FastAPI 앱을 Render.com에 배포했는데 [증상 설명]

[Render 로그]
(Deploy 또는 Runtime 로그에서 에러 부분 복사)

[프로젝트 구조]
(파일/폴더 구조 간단히)

[설정]
- Start Command: uvicorn app.main:app --host 0.0.0.0 --port $PORT
- Python 버전: 3.11
- 환경변수: DATABASE_URL, SECRET_KEY 설정됨

로컬에서는 정상 동작합니다. 배포에서만 에러가 납니다.
```

### 템플릿 3: "이거 어떻게 하는 건가요?"

```
FastAPI + 바닐라 HTML 프로젝트에서 [하고 싶은 것]을 구현하고 싶습니다.

[현재 코드]
# 관련 코드

[원하는 결과]
- [구체적으로 어떤 동작을 원하는지]

이 기능을 구현하는 코드를 작성해주세요.
```

## 에러 메시지 복사하는 방법

### 브라우저 Console에서

```
1. F12 → Console 탭
2. 빨간 에러 메시지 위에서 마우스 우클릭
3. "Copy" → "Copy message" 또는 에러 텍스트 드래그하여 복사
```

### 브라우저 Network에서

```
1. F12 → Network 탭
2. 실패한 요청 클릭
3. Response 탭의 내용을 전체 선택(Cmd+A) → 복사(Cmd+C)
```

### 서버 터미널에서

```
1. Traceback 전체를 드래그하여 선택
2. 복사 (Cmd+C)
3. 최소한 마지막 3-5줄은 포함할 것
```

### Render 대시보드에서

```
1. Logs 탭에서 에러가 나온 부분 확인
2. 해당 줄을 드래그하여 복사
3. Deploy 로그인지 Runtime 로그인지도 함께 알려줄 것
```

## AI 답변을 적용할 때 주의할 점

### 1. 코드를 통째로 바꾸지 마라

```
AI가 제안한 코드 중 변경된 부분만 적용한다.
전체를 복사 붙여넣기 하면 기존에 잘 되던 다른 코드까지 날아갈 수 있다.
```

### 2. 한 번에 하나씩 적용하라

```
AI가 3가지를 고치라고 했으면:
1. 첫 번째만 고치고 테스트
2. 두 번째만 고치고 테스트
3. 세 번째만 고치고 테스트

→ 어떤 수정이 효과가 있었는지 알 수 있다
```

### 3. 이해하고 적용하라

```
AI 코드를 그냥 붙여넣지 말고:
- "이 코드가 뭘 하는 건지" 한 줄씩 읽어본다
- 모르는 부분이 있으면 "이 코드에서 xxx 부분이 뭘 하는 건지 설명해줘"라고 추가 질문
- 이해하고 적용해야 다음에 비슷한 에러가 나도 스스로 해결할 수 있다
```

### 4. 후속 질문을 활용하라

```
첫 답변이 안 통하면:
"위 방법을 적용했는데 이번에는 다른 에러가 납니다: [새 에러 메시지]"

맥락이 유지되므로 더 정확한 답을 받을 수 있다.
```

---

## 핵심 정리

1. **에러 메시지를 반드시 포함**해서 질문한다 (복사 붙여넣기)
2. **관련 코드**를 함께 보여준다 (서버 + 프론트 양쪽)
3. **시도한 것**을 알려주면 중복 답변을 피할 수 있다
4. AI 답변은 **한 번에 하나씩**, **이해하면서** 적용한다
5. 안 되면 **새 에러 메시지와 함께** 후속 질문한다
